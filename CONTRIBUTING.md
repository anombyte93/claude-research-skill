# Contributing to Research Before Coding

Thanks for your interest in improving this skill! This guide will help you get started.

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/anombyte93/claude-research-skill/issues) to report bugs or suggest features
- Check existing issues before creating a new one

### Submitting Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Test your changes with Claude Code
4. Commit with clear messages
5. Push to your fork and submit a Pull Request

### Areas for Contribution

- Better research query templates
- Additional synthesis patterns
- Documentation improvements
- Integration with more MCP tools

### Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/claude-research-skill.git

# The skill is a standalone .md file - no build step required
# Test by copying to ~/.claude/skills/research-before-coding/
```
